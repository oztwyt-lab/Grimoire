# Grimor Privacy Policy Draft

Last updated: 2026-05-10

Grimor is a recipe, pantry, and cooking companion app by Slingblade Studios.

This draft is intended for Play Store/internal testing preparation and should be reviewed before public launch.

## Data We Collect

Grimor may collect:

- Account information: email address and Firebase user ID.
- Profile information: nickname and selected character.
- User content: recipes, recipe ingredients, recipe steps, inventory items, expiry dates, equipment selections, and feedback messages.
- App settings: language, sound settings, recent recipe IDs, and local shopping list data.
- Technical metadata: app version, platform, timestamps, and crash/error details shown locally by the app.

## How We Use Data

We use this data to:

- Create and manage user accounts.
- Save recipes, inventory, equipment, and preferences.
- Show matching recipes and cooking flows.
- Provide customer support and review feedback.
- Maintain app security, troubleshoot issues, and improve Grimor.

## Data Storage

Account, recipe, inventory, profile, equipment, subscription-state, and feedback data are stored with Firebase services.

Some data is stored only on the user's device through local storage, such as language preferences, recent recipe IDs, sound preferences, and the shopping list.

## Data Sharing

We do not sell user data.

We use service providers, including Firebase/Google Cloud, to host authentication and app data. These providers process data on our behalf.

## Account Deletion

Users can delete their account from the app settings. Account deletion removes the Firebase Auth account and associated Grimor user data, including recipes, profile/inventory data, and feedback entries linked to the user ID.

## Security

Grimor uses Firebase Authentication and Firestore security rules to restrict user data access. Users should keep their account password secure.

## Children's Privacy

Grimor is not intended for children under 13. Do not use Grimor if you are under 13.
